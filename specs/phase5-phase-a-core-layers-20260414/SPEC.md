# 日本房价地图 Phase 5 Phase A Core Layers SPEC

## 1. 目标

在当前单页东京地图底座上，连续落地前三类核心图层：

- `price / land`
- `schools / convenience`
- `hazard / population`

这轮不是继续做 seed demo，而是把这六个模式都接到正式数据口径，并让地图壳、图层 schema、站点面板、README 一次同步到可解释状态。

## 2. 本轮范围

### 必做

- 新增 `convenience` 和 `population` 两个正式模式
- 把前端数据模型升级成：
  - 站点层
  - 点位层
  - 区域层
- 生成新的东京运行时数据文件：
  - `stations.json`
  - `schools.json`
  - `convenience.json`
  - `hazards.json`
  - `population.json`
- 让地图在六个模式之间即时切换
- 让站点卡片能展示六个模式的正式摘要

### 这轮选定的数据口径

#### `price`

- 来源：不动产信息库站点搜索页背后的官方结果接口
- 口径：
  - 东京都
  - `2024Q1 ~ 2024Q4`
  - `transactionPrice = true`
  - `kind = residential`
- 聚合单位：
  - 以结果里的 `station_name_ja` 直接回挂到官方车站主表
  - 取站点近邻成交样本的总价中位数
  - 同时计算面积口径的中位单价
- 限制：
  - 这是正式成交价结果，不再沿用手写种子
  - 但构建脚本不把 subscription key 写进仓库

#### `land`

- 来源：国土数值信息 `L01-25`
- 口径：
  - 2025 年地价公示
  - 点位按最近站聚合
- 聚合单位：
  - 站点 1.2km 内最近归属
  - 取公示点单价中位数

#### `schools`

- 来源：国土数值信息 `P29-23`
- 口径：
  - 东京都学校点
  - 点位按最近站归属

#### `convenience`

- 来源：
  - 国土数值信息 `P04-20` 医疗机构
  - 国土数值信息 `P05-22` 公共施設
- 口径：
  - 先做一版“医疗 + 公共服务”的官方便利度代理
  - convenience score 为组合分，不伪装成绝对真值

#### `hazard`

- 来源：国土数值信息 `A31a-24`
- 口径：
  - 使用 `想定最大規模`
  - 当前正式接入第一版洪水浸水风险
- 区域裁剪：
  - 只保留真正覆盖到站点的风险面

#### `population`

- 来源：国土数值信息 `500m_mesh_2024_13`
- 口径：
  - 500m mesh 将来推计人口
  - 以 `2020 -> 2040` 变化率定义增长 / 稳定 / 收缩
- 区域裁剪：
  - 只保留包含当前东京核心站点的 mesh

## 3. 不做什么

- 不做独立详情页
- 不做 AI 功能
- 不扩到东京以外城市
- 不把便利度包装成“完整生活评分体系”
- 不把灾害风险包装成“全灾种完成版”，这轮先正式化洪水风险

## 4. 关键原则

- 这是地图工具，不做内容站。
- 用户一进来就是东京地图。
- 模式切换只做成按钮切换，不新增页面层级。
- 所有图层都围绕车站锚点联动。
- 对没有覆盖的数据，明确显示未覆盖，不补假值。
- 对“官方但非完美”的代理指标，要把口径写清楚。

## 5. 验收

- 六个模式在主地图里都能切换
- `price` 不再是手写 seed，而是正式成交价聚合结果
- `land` 是正式公示地价聚合结果
- `schools / convenience` 都是正式点位图层
- `hazard / population` 都是正式区域图层
- 站点卡片能解释六个模式的核心字段
- `README.md` 写清楚：
  - 数据源
  - 时间口径
  - 覆盖率
  - 当前边界
