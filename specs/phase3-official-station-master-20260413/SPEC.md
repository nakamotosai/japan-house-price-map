# 日本房价地图 Phase 3 Official Station Master SPEC

## 1. 目标

把当前东京地图里“手写站点坐标 + 手写热度”的做法，升级成“官方车站主表 + 官方站别客流 + 前台可直接使用的东京车站底座”。

这一轮重点是把地图底座做实，而不是一次把所有模式都官方化。

## 2. 范围

- 接入国土数值信息 `N02-24` 铁道站点主表
- 接入国土数值信息 `S12-24` 站别客流数据
- 生成东京核心范围的官方车站主表
- 用真实客流重算 `ridershipDaily`、`heatScore`、`transferLines`、默认大站标签
- 现有手写 `stations.json` 改成“种子覆盖层 + 生成产物”
- 前台兼容“有些站已经有官方坐标和客流，但价格/地价/灾害仍未覆盖”的状态

## 3. 交付标准

- 地图上的车站坐标不再主要依赖手写 JSON
- `heat` 模式的站点热度和客流来自官方 `S12-24`
- 默认显示的站点总数显著增加，且仍保持可用
- 大站标签逻辑不再只靠手写名单，而是结合真实客流和换乘线数
- 对于暂未覆盖价格/地价/灾害的站点，前台显示中性占位，不展示假数字
- 导入流程可以重复运行，重新生成 `public/data/tokyo/stations.json`
- 测试、lint、build、预览验证通过

## 4. 非目标

- 不在这一轮接入正式房价成交流水线
- 不在这一轮接入正式公示地价图层
- 不在这一轮做学校、灾害的全量官方替换
- 不在这一轮做站点详情页和 AI
